Fundamental Web Development Concepts

  1. Demonstrate knowledge of HTML and CSS
  2. Identify PHP programing concepts
  3. Identify JavaScript and jQuery programing concepts
  4. Demonstrate the use of Git for version control

You are building a solution in PHP using object orientated methodologies. You want to be able to create variations of a base class and the base class should never be instantiated. What do you do?

  1. Create an abstract base class and extend it with each variation.
  2. Create an interface base class and extend it with each variation.
  3. Create an individual class for each variation.
  4. Write the functionality using procedural programming in individual files and include them when necessary.

There are some lines in custom code that are causing an issue. The code is version controlled with Git.
You want to figure out who wrote the lines of offending code. How do you it?

  1. Use the git log command and view who last committed to the file.
  2. Use the git log command with the grep option to search for the offending lines of code.
  3. Use the git blame command and view the user who committed the lines.
  4. Send an email to the team to ask who added the offending lines of code.

Site Building

  1. Demonstrate ability to create and configure Content Types with appropriate fields and field settings for building basic data structures
  2. Demonstrate ability to configure field display and view modes for content types
  3. Demonstrate ability to create and use Taxonomy vocabularies and terms for classification and organization of content
  4. Demonstrate ability to configure Blocks for building layouts from information widgets
  5. Demonstrate ability to build main and alternative navigation systems by using Menus
  6. Demonstrate ability to create and configure Views for building content list pages, blocks and feeds

You have 2 content types Blog and Article, an image field is used on both.
The site administrator wants to remove the image field from the Article content type.
Which statement below is true?

 

  1. Deleting the field instance on the Article content type won't affect Blog node content.
  2. You cannot delete a field once it has stored data.
  3. Delete the field instance will remove it from both the Article and Blog content types.
  4. Deleting the Article field instance will only affect new Article nodes, the old nodes will still display the field.

Front end development (theming)

  1. Given a scenario, demonstrate ability to create a custom theme or sub theme
  2. Demonstrate knowledge of theming concepts
  3. Demonstrate ability to build or override PHP templates for defining layout content
  4. Demonstrate ability to use theme () functions for overriding custom output
  5. Demonstrate ability to write template pre-process functions for overriding custom output

You have a content type Blog. The client wants a page displaying a Views list of the 10 most recently created Blog nodes. The list must display a stripped down version of the full Blog node. Which one of the following solutions should you use?

  1. Copy the views template files into the theme and update them to strip out the desired fields.
  2. Use the views display format to display the full Blog content then update the node template file to check for the current page to display the stripped down content.
  3. Use the fields display format in Views to select only the fields to display.
  4. Configure the teaser display mode for the Blog content type and update the views display format to use the new display mode.

Domain 4.0: Back end development (coding)

  1. Demonstrate ability to develop Custom Modules using Drupal API for extending Drupal functionality
  2. Demonstrate ability to work with Drupal's Database Abstraction Layer for managing tables and CRUD operations on data
  3. Demonstrate ability to debug code and troubleshoot site problems
  4. Demonstrate ability to write code using Drupal Coding Standards
  5. Demonstrate ability to analyze and resolve site performance issues arising from site configuration and custom code
  6. Demonstrate ability to analyze and resolve security issues arising from site configuration and custom code

You are working on a client website. A custom module; key_data has been developed and needs to be updated. A new table required to store additional meta data. The module is already live, how do you update it?

  1. Implement key_meta_data_install() to specify the new table schema. Deploy the changes and run update.php on the site.
  2. Update the function key_meta_data_schema() implementation with the new table definition, write the function key_meta_data_update_N() that calls the function drupal_install_schema(). Deploy the changes and run update.php on the site.
  3. Update the existing function key_data_schema() with the new table definition. Implement key_data_update_N() and call the function db_create_table(). Deploy the changes and run update.php
  4. Log into mysql on the live server and run the relevant commands to create the new table.
